# 前端联调自测清单（web-agent / `index.html`）

本文档汇总 **当前阶段建议在后端就绪后执行的联调自测**（不替代自动化测试）。执行时在每条前打勾即可。

**权威契约**：A-Agent **`note/API接口总表.md`**（含附录 JSON 样例）+ **`note/前端与用户指引.md`** + **`note/导演台与叙事功能使用说明.md`**（M1～M10、yml、导演台步骤）。本仓库备忘见 `前端对接需求.md`。

**前置**：配置 `axios` 的 `baseURL` 指向联调后端（小说域默认 `/api/novel`）；若开启安全，先完成管理员登录再测非公开书。

---

## 1. `GET /api/novel/config/frontend-runtime`

| # | 步骤 / 期望 | 结果 |
|---|----------------|------|
| 1.1 | 启动应用或触发 `fetchFrontendRuntime`（如打开创建书弹窗、`saveConfig` 成功后），请求成功 | [ ] HTTP 200，前端日志出现「已同步 frontend-runtime」且含 **M7、M9** 开关与 `autoContinueMax` |
| 1.2 | 核对响应字段 **`m9CrosscutEnabled`**：为 `true` 时与 yml `m9-crosscut-enabled` 一致 | [ ] |
| 1.3 | **`m7ArtifactEnabled`** 为 `false` 时：「质量与调试」内不应出现「生成快照」Tab；若当前 Tab 为生成快照，应自动回到「导出体检」 | [ ] |
| 1.4 | **`m9CrosscutEnabled`** 为 `false` 时：「质量与调试」内不应出现「跨章快照」Tab；若当前 Tab 为跨章快照，应自动回到「导出体检」 | [ ] |
| 1.5 | **`autoContinueMaxTarget`**：自动续写表单的提示文案中的上限与响应一致；数值超过后端上限时提交应被前端 clamp 或后端拒绝（与现实现一致） | [ ] |
| 1.6 | **`autoContinueDefaultTarget`**、大纲相关 min/max/default（若 UI 暴露）与响应一致 | [ ] |
| 1.7 | 故意使该接口失败（断网或错误 URL）：前端有警告日志，自动续写上 fallback 为本地默认 clamp，应用不崩溃 | [ ] |
| 1.8 | **`jwtExpirationMs`**（若展示）与登录响应 **`expiresInMs`** 一致（见 `前端对接需求` §2） | [ ] |

## 2. M9：`GET /api/novel/{novelId}/narrative-state`

**前提**：`frontend-runtime` 中 **`m9CrosscutEnabled === true`**（否则本组可标 N/A）。

| # | 步骤 / 期望 | 结果 |
|---|----------------|------|
| 2.1 | 选中一本书 → 打开「质量与调试」→ 切到「跨章快照」，应发起 `GET .../narrative-state` | [ ] 网络面板可见请求 |
| 2.2 | 成功且 body 为 **`{ data: { ... } }`** 或 **扁平 Map**：前端能解析 **`novelId`、`narrativeState`、`message`、`narrativeStateRaw`**（实现使用 `res.data?.data \|\| res.data`） | [ ] 界面展示 JSON 或空态/提示与总表样例一致 |
| 2.2b | **`narrativeState` 非 null** 且后端已开 M9 v1.1：JSON 中出现 **`relationshipHints` / `tensionRippleHint` / `schemaVersion`**（与附录 A.1 一致） | [ ] |
| 2.3 | **`narrativeState` 为 null** 且无文案：显示「尚无跨章快照」类空态 | [ ] |
| 2.4 | 仅有 **`message`**、无 JSON 可展示：出现 `el-alert` 展示 `message` | [ ] |
| 2.5 | 点击「质量与调试」底部「刷新当前标签」且当前为跨章快照 Tab：`loading` 生效，数据刷新 | [ ] |
| 2.6 | 接口 4xx/5xx 或网络错误：错误提示「获取跨章快照失败」，bundle 为已加载但内容清空（与现实现一致） | [ ] |

## 3. 「质量与调试」Tab 与 M7

| # | 步骤 / 期望 | 结果 |
|---|----------------|------|
| 3.1 | **`m7ArtifactEnabled === true`**：存在「生成快照」Tab；切换至该 Tab 时拉取 `GET .../narrative-artifacts` | [ ] |
| 3.2 | M7 关闭时：无「生成快照」Tab；从 `fetchFrontendRuntime` 刷新后若曾停在该 Tab，应切回「导出体检」 | [ ] |
| 3.3 | 各 Tab（导出体检、一致性告警、章节事实、主线快照、生成快照、跨章快照）与「刷新当前标签」组合无 JS 报错 | [ ] |
| 3.4 | **`GET .../pipeline`**（总表 §5）：响应含 **`hotMemeEnabled`、`libraryPublic`** 时，选书或打开「文风」后，故事信息里的热梗开关、列表上的书库可见标签与接口一致 | [ ] |

## 6. 其它已对接能力（回归抽查）

以下详见 `前端对接需求.md` §3 与总表；联调窗口内建议至少各抽 1 条冒烟。

| 能力 | 建议最小验证 | 结果 |
|------|----------------|------|
| 列表 / 详情 / 章节 / 进度 / pipeline | 选书后章节列表与进度展示正常 | [ ] |
| 创建书 | `POST /create` 或文风创建 URL 与 `writingPipeline` dash 别名一致 | [ ] |
| 单章续写 | **文风入口**：`POST /api/novel-style/{style}/{novelId}/continue`；**小说域**（与导演台一致）：`POST /api/novel/{id}/continue`；二者任务队列语义一致 | [ ] |
| 自动续写 | `POST /{id}/auto-continue`，目标章范围与 runtime 上限一致 | [ ] |
| 文风微参 M6 | `POST /{id}/writing-style-params`（`narrative`、`narrativePhysicsMode`） | [ ] |
| 大纲再生与任务 | `POST /{id}/outline/regenerate`、`GET /{id}/generation-tasks`、cancel/kick/retry 规则 | [ ] |
| 流水线保存 | `POST /{id}/pipeline`，Body dash 别名 | [ ] |
| 书籍备注 | 仅改连载平台或仅改创作说明时，请求体**只含变更字段**（与《前端与用户指引》）；无变更时前端提示「书籍备注无变更」且不 POST | [ ] |
