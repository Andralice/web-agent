## 2. 环境与鉴权（请后端填写）

| 字段 | 示例 | 说明 |
|------|------|------|
| 默认 Base URL | `http://localhost:8080` | 与 `server.port` 一致 |
| 生产 / 测试 Base URL | （部署时填写） | 多环境时逐条列出 |
| `app.security.enabled` | `true`（默认 yml） | 为 `true` 时须登录管理员 JWT 才能读 **`libraryPublic=false`** 的书 |
| 管理员账号（仅联调用） | 用户名见 `app.security.admin-username`；**密码勿提交 git**，生产用环境变量覆盖 `app.security.admin-password` | 默认 yml 含本地明文密码，仅内网 |
| JWT 过期时间 | **`expiresInMs`**：`86400000`（24h，与 `app.security.jwt-expiration-ms` 一致） | 登录成功响应字段 **`expiresInMs`**（见 `POST /api/auth/login`）；亦可 **`GET /api/novel/config/frontend-runtime`** 读 `jwtExpirationMs` |

**前端行为假设（与后端一致）：**

- Map 形态成功：`status=success` 且 `code=OK`。
- 失败：展示 `message`；`code` 用于分支（如 `TASK_CONFLICT`、`STYLE_CONTINUE_TASK_FAILED`）。
- 非公开书：访客可能收到 **404**（非 403），前端应提示「需登录或无权限」，而非「小说不存在」。

## 3. 前端已对接的 HTTP 面（摘要）

以下与 `index.html` 中 `axios` 的 `baseURL` 组合使用（小说域默认 ` /api/novel`）。

| 能力 | 方法路径（相对 novel base） | 备注 |
|------|------------------------------|------|
| 列表 / 详情 / 章节 / 大纲 / 进度 / pipeline | `GET /list`、`GET /{id}`、`GET /{id}/chapters` 等 | 详情含 `libraryPublic`、`narrativeCarryover` 等 |
| 创建 | `POST /create` 或 `POST /api/novel-style/{style}/create` | 文风创建走独立 base |
| 单章续写（二选一） | **`POST /api/novel-style/{style}/{novelId}/continue`**（默认）或 **`POST /{novelId}/continue`**（小说域） | 前者 `{style}` 须与 `fromPath` 一致，响应可有 **`pipelineHint`**；后者与 **同源导演台** `novels.html`「续写一章」同路径；**实际写作**均以书本 `writingPipeline` 为准（总表「文风与流水线」） |
| 自动续写 | `POST /{id}/auto-continue` | 仍走 novel base |
| 文风微参 | `POST /{id}/writing-style-params` | Body：`writingStyleParams`（对象）或 **`writingStyleParamsRaw`**（字符串）；根级可含 **`narrative`**、**`narrativePhysicsMode`**（M6）。当前 UI 仅 POST 对象形态 |
| 大纲再生 | `POST /{id}/outline/regenerate` | 同步长请求；提交前会静默拉任务列表 |
| 任务 | `GET /{id}/generation-tasks`，`POST /api/novel/tasks/{taskId}/cancel|kick|retry` | `OUTLINE_REGENERATE` 不 kick |
| M7 侧车 | `GET /{id}/narrative-artifacts` | 质量与调试 ·「生成快照」Tab |
| M9 跨章叙事 | **`GET /{id}/narrative-state`** | 书本级 `narrativeState` 聚合；依赖 yml **`m9-crosscut-enabled`** 与总表 §1.3；**`m9CrosscutEnabled=false` 时隐藏**「跨章快照」Tab。 |
| 流水线保存 | `POST /{id}/pipeline` | Body 使用 **dash 别名**（见 §4.1） |
| 书籍备注 | `POST /{id}/book-meta` | **`serializationPlatform` / `creatorNote` 未出现在 body 则不修改**；清空传 `""`。实现见 `submitBookMeta`（与《前端与用户指引》一、16.3 一致） |
| 创作全过程（可选） | **`GET /{id}/creative-process`** | 聚合视图；时间线 `novel_created` 含书籍备注。产品 UI 未接时可仅用总表 §15 对照 |
| **运行时配置** | **`GET /api/novel/config/frontend-runtime`** | 无需登录；clamp、JWT 时长、M7/**M9** 开关等。**前端已实现**：启动、`saveConfig`、打开创建书弹窗成功后拉取并合并 clamp、`autoContinueMaxTarget`；`m7ArtifactEnabled=false` 时隐藏「生成快照」Tab；**`m9CrosscutEnabled=false` 时隐藏「跨章快照」Tab** 并避免卡在 M9。 |

## 4. 需要后端以「机器可读表」回填的数据（核心）

### 4.1 `WritingPipeline.fromPath` 允许输入全集

前端将 **`writingPipeline`（实体）** 与 **`currentPipeline`（GET pipeline）** 映射为 **`/api/novel-style/{style}/...` 的 `{style}`**，并向 **`POST .../pipeline`** 提交 **`pipeline` 字符串**。**官方允许列表**如下（大小写不敏感；**未列出**的任意字符串**静默**解析为 **`POWER_FANTASY`**，HTTP 通常仍为 200）。

| 枚举名（落库 / GET 常见值） | 推荐 URL / Body 字符串（dash） | 是否默认爽文 | 其它已识别别名（节选） |
|------------------|----------------------------------|:--:|------|
| `POWER_FANTASY` | **`power-fantasy`**（前端「默认爽文」推荐） | 是 | `power_fantasy`、`powerfantasy`、`pf`、`default`、`shuang`、`shuangwen`、`爽文` |
| `LIGHT_NOVEL` | **`light-novel`** | 否 | `light_novel`、`lightnovel`、`light`、`ln` |
| `SLICE_OF_LIFE` | **`slice-of-life`** | 否 | `slice_of_life`、`sliceoflife`、`slice`、`daily` |
| `PERIOD_DRAMA` | **`period-drama`** | 否 | `period_drama`、`perioddrama`、`period`、`age`、`era`、`年代`、`年代文` |
| `VULGAR` | **`vulgar`** | 否 | `rough`、`俗`、`粗俗`、`粗口` |

**前端当前映射逻辑（供对照）：** 见 `index.html` 内 `novelStyleUrlSegmentFromWritingPipeline`、`pipelineValueForApi`、`novelStyleUrlSegment`（新建书文风 URL）。若上表与实现不一致，**以前端收到此表为准**改代码。

### 4.2 错误响应样例（每种至少 1 个真实 JSON）

以下与 **A-Agent `API接口总表.md` 附录 B～G** 一致；HTTP 状态与 `ApiResponse` / Map 形态见该附录说明。

| 场景 | HTTP | 响应 body 样例 |
|------|------|----------------|
| `TASK_CONFLICT`（续写） | 200（Map） | `{"status":"error","code":"TASK_CONFLICT","message":"目标章节正在重生/续写中，请稍后重试。"}` |
| `TASK_CONFLICT`（大纲再生） | 200（Map） | `{"status":"error","code":"TASK_CONFLICT","message":"本书存在进行中的章节生成任务，请等待结束后再修改大纲"}` |
| `STYLE_CONTINUE_TASK_FAILED` | 200（Map） | `{"status":"error","code":"STYLE_CONTINUE_TASK_FAILED","message":"服务暂时不可用，请稍后重试"}`（`message` 为脱敏文案，以实际为准） |
| `INVALID_STYLE_PARAMS` | 200（Map） | `{"status":"error","code":"INVALID_STYLE_PARAMS","message":"writingStyleParams 不是合法 JSON 或不包含支持的字段"}` |
| 访客访问 `libraryPublic=false` 的书 | **404** | `{"code":404,"message":"未找到该小说，请检查链接或刷新列表","data":null}` |
| 未登录调 `library-visibility` | **403** | `{"code":403,"message":"需要管理员登录","data":null}` |

### 4.3 `GET /api/novel/{id}/generation-tasks` 单条任务 JSON 样例

完整字段列表与三条 `taskType` 示例见 **A-Agent API 附录 H～J**。摘要：

| taskType | 用途 | 样例要点 |
|----------|------|-----------|
| `CONTINUE_SINGLE` | 章节 worker | `rangeFrom`=`rangeTo`=目标章；`payloadJson` 含 `chapterNumber` |
| `OUTLINE_REGENERATE` | 大纲占位租约 | `rangeFrom`/`rangeTo`/`currentChapter` 多为 `0`；`status` 常为 `RUNNING` 直至结束 |
| `INITIAL_BOOTSTRAP` | 开书 | `rangeFrom`=1，`rangeTo`=5（初版引导）等，以库内为准 |

**字段约定**：`rangeFrom` / `rangeTo`、`currentChapter`、`status`（`PENDING|RUNNING|DONE|FAILED|CANCELLED`）、`taskType`、`id` 与 ORM 实体一致，与表格列展示一致。

### 4.6 运行时配置（与 UI 校验相关）

**首选**：启动后请求 **`GET {apiBase}/api/novel/config/frontend-runtime`**，与 UI 对齐（**无密钥**）。

| 配置键（响应字段名） | 默认仓库 yml 典型值 | 前端用途 |
